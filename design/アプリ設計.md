# Mandalart App 設計書（アプリ設計 → LPに必要な要素 → LP制作）

> **この設計書について**
> **目的は、Mandalart App のサービス紹介LPを作ること。**
> そのために本書では、まず `§1〜§5` でアプリの作りと良い機能を整理し、`§6` でそこから **「LPに必要な要素」を引き出して、LP制作の入力にする**。
> ＝「アプリ設計 → LPに必要な要素 → LPを作る」という順で読む。
> 見た目（配色・タイポグラフィ等）は本書では定めず、別紙 **`DESIGN.md`（デザイン指定）** に従う。

## 1. プロダクト概要・コンセプト

- **一言で**：個人の目標（マンダラート）を、1on1・タスク・経歴・分析とひと続きにし、「目標の立てっぱなし／1on1のやりっぱなし」をなくす**組織成長プラットフォーム**。
- **統合の思想**：バラバラのツール（目標はExcel、1on1はメモ、タスクは別アプリ…）に散った「人の成長に関わるデータ」を1つに束ね、**目標 → 実行（タスク）→ 対話（1on1）→ 振り返り（分析）→ 蓄積（経歴）** のループを1プロダクトで回す。
- **なぜマンダラートか**：3×3を9つ並べた **9×9＝81セル**のフレームワーク。中心の大目標を8つの中目標へ、さらに64の具体アクションへ分解する。大谷翔平選手の活用例で知られ、**抽象的な目標を「毎日の行動」レベルまで具体化できる**のが核。これをプロダクトの背骨に据える。

## 2. ターゲットと、潰しにいく課題

**プロダクトが解決する3つの課題：**

1. **目標の形骸化** — 期初の目標がExcelに埋もれ、振り返りが形式的。成長実感が薄い。
2. **1on1の属人化・散在** — マネージャーごとに進め方がバラバラ、記録が散らばり、異動で引き継げない。
3. **ツールの分散** — 目標・1on1・タスク・経歴が別ツールで、データが連携せず全体像が掴めない。

**利用者（3階層）：**

| 階層 | 主な使い方 | 得たい結果 |
| ---- | ---- | ---- |
| 人事・経営 | 組織横断で達成・1on1実施・コンディションを把握 | データに基づく人材育成戦略、経営報告 |
| マネージャー | チーム全員の状態・次回1on1・タスク進捗を把握 | 的確なフォロー、1on1準備の時短 |
| メンバー | マンダラートで目標を構造化、月次で振り返り | 成長実感、迷いのない行動 |

**想定組織規模**：〜50名／51〜300名／301名以上（料金プランに対応）。

## 3. アプリの作り（機能モジュール構成と連携）

### 6つの機能モジュール

1. **マンダラート目標管理（中核）** — 81セルで目標を構造化・可視化
2. **1on1ミーティング管理** — 計画・実施・振り返りの一気通貫
3. **カンバンタスク管理** — 実行をボードで可視化
4. **経歴・スキル管理** — メンバーのスキル・資格・経験の一元管理
5. **お知らせ・通知** — 組織内の情報伝達
6. **成長分析・ダッシュボード** — 全データの横断可視化

### 連携の肝（＝「ひと続き」にする設計）

- **目標・タスクが1on1画面に統合表示** → 1on1を「目標と実行の進捗を見ながら」行える（対話が具体になる）。
- **全モジュールのデータを分析が横断** → 個人／部署／全社の成長を同じ物差しで可視化。
- **経歴・スキルが配置・アサインの土台** → 「誰が何をできるか」を AIベクトル検索で引ける。
- この連携こそが「課題3：ツール分散」への直接の回答。単機能の寄せ集めではなく、**目標起点でデータが繋がる**ことが価値の中心。

### 基盤・技術

- **クラウド/認証**：Microsoft Azure 運用、Azure AD（MSAL）認証、SSO、HTTPS、ロールベースアクセス制御（RBAC）。
- **AI**：1on1アイスブレイクテーマの自動提案（Gemini API）／新規技術要素のカテゴリ自動判定／AIベクトル検索によるスキルマッチング。
- **外部連携**：Outlookカレンダー連携（1on1予定）。
- **対応端末**：レスポンシブ（PC／タブレット／スマートフォン）。

### データ入出力

- **入力**：Excel経歴書インポート（マージセル対応・名寄せ処理付き）／CSV目標インポート。
- **出力**：マンダラートのPNG画像出力／レーダーチャート付き経歴PDF出力。

## 4. 主要機能と強み（良い機能）

| 機能 | 中身 | 強み（なぜ良いか） | 対応する課題 |
| ---- | ---- | ---- | ---- |
| **マンダラート目標管理** | 81セルで大目標→中目標→アクションに分解／4段階の達成レベル（意識し始めた→常に意識できている）／月別の目標・実績管理／達成率リアルタイムゲージ／過去月比較の成長ハイライト | 目標が「毎日の行動」まで具体化され、**形骸化しない**。達成度が常に可視 | 課題1 |
| **1on1ミーティング管理** | 事前メモ・テーマ設定・感想記入・履歴管理／AIアイスブレイク提案／Outlook連携／コンディションのダッシュボード一覧／テーマ別履歴の円グラフ分析 | 進め方を標準化して**属人化を解消**、履歴が残るので**異動でも引き継げる** | 課題2 |
| **カンバンタスク管理** | 未着手・進行中・完了の3ステータス／カスタムラベル・優先度／Markdown説明欄／**1on1画面との統合表示** | 実行が見える化され、**1on1・目標と地続き**になる | 課題2・3 |
| **経歴・スキル管理** | ポジション別・技術別の経験分析／Excel経歴書インポート／レーダーチャート付きPDF／**AIベクトル検索**でメンバー発見 | スキルの棚卸しと**人材発見・配置**ができる | 課題3 |
| **お知らせ・通知** | Markdownエディタ（プレビュー付き）／未読・既読フィルター／通知ベル連動 | 重要情報を**確実に届ける** | 課題3 |
| **成長分析・ダッシュボード** | 月別達成率の折れ線／部署横断の達成状況チェック／年度間データ比較 | **データ駆動**で組織の成長を捉え、育成判断に使える | 課題1・3 |

> **差別化の核**：①「マンダラート」という独自の目標構造、② 目標起点で全データが繋がる統合性、③ 育成文脈に効くAI（アイスブレイク・カテゴリ判定・ベクトル検索）。この3点が「良い機能」群の背骨。

## 5. 提供価値（立場別の便益）

- **人事・経営**：部署横断で達成状況・1on1実施率・コンディションを一元管理し、データに基づく人材育成戦略を立案できる。経営会議への報告がスムーズになる。
- **マネージャー**：メンバーのコンディションと次回1on1・タスク進捗を一画面で把握し、的確にフォローできる。1on1の事前準備が短縮される。
- **メンバー**：目標が81セルに分解され、毎日何をすべきか迷わない。月次の振り返りと成長ハイライトでモチベーションが続き、経歴も自動で蓄積される。

## 6. ここから導く「LPに必要な要素」

> `§1〜§5`（アプリの設計）から、**LPで何を・どの順で・誰に向けて訴求するか**を導く。
>
> **ここでLPの文言（コピー）そのものは固定しない。** 見出しや本文の文章まで指定すると、生成結果が一通りに決まり、コピーもデザインも起こす余地がなくなる。本書が決めるのは「**どの強みを・どんな狙いで・どの順に載せるか**」まで。実際の文章とレイアウトは制作時に起こす前提なので、仕上がりは人によって変わってよい。

### 6-1. LPのゴール

- **問い合わせ・相談の獲得**（リード獲得）。一次CTAは一貫して「相談／お問い合わせ」へ寄せる。

### 6-2. 訴求の優先順位（アプリの強みから）

1. **統合＝ツール分散の解消**（全体の価値・`§3`の連携） … LP全体を貫く軸。
2. **マンダラート目標管理**（差別化の主役・`§4`） … 一番に押す独自性。
3. **1on1管理**（2番目の主役・`§4`） … 「やりっぱなしをなくす」。
4. **AI・連携・セキュリティ**（信頼の裏付け・`§3`） … 検討者の不安を消す。

### 6-3. ターゲット別に伝えるべき要点（文言は生成に委ねる）

- 人事・経営へ：組織全体の成長が数字で見えること。
- マネージャーへ：チームの状態が一望でき、1on1準備が軽くなること。
- メンバーへ：目標が行動レベルまで分解され、迷わなくなること。

### 6-4. LPに必要なセクション（各セクションの狙いと根拠）

| LPセクション | そのセクションで達成すること | 根拠（本書の対応箇所） |
| ---- | ---- | ---- |
| ヒーロー | 統合コンセプトと主役（マンダラート）を一目で伝え、信頼の数字（機能数・セル数・AI搭載）を添える | `§1` コンセプト |
| お悩み（3つ） | ターゲットの課題を自分ごと化させる | `§2` 課題 |
| 機能一覧（6つ） | 6モジュールの全体像を見せる | `§3`・`§4` |
| 機能ハイライト①：マンダラート | 差別化の主役を深掘り（81セル分解・4段階達成・月別/比較） | `§4` 主役機能 |
| 機能ハイライト②：1on1 | 第2の主役を深掘り（事前メモ〜履歴・AIアイスブレイク・Outlook連携） | `§4` |
| 利用シーン（人事/マネージャー/メンバー） | 立場別の便益に翻訳する | `§5` 提供価値 |
| 画面イメージ | UIの具体像で安心感を与える | `§3` 機能群 |
| 料金（3プラン） | 規模別の選択肢を示す（中央を強調） | `§2` 規模 |
| FAQ | 導入・セキュリティ・AI等の不安を解消 | `§3` 基盤・入出力 |
| CTAバナー＋お問い合わせ | 最終ひと押しと、規模を取得するフォーム | `§6-1` ゴール |

### 6-5. CTA・導線

- ヘッダー／ヒーロー／CTAバナー／フォームへ一次CTAを反復配置。フォームで従業員規模を取得し、規模別プランへ接続する。

### 6-6. 見た目・トーン

- 配色・タイポグラフィ・余白・角丸・影などの**見た目は本書では定めない**。別紙 **`DESIGN.md`** に従う。
